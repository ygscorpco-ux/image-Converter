# PNG 로고 파생본 자동 생성기 개발계획서

## 1. 프로젝트 개요

### 1-1. 프로젝트명
PNG 로고 파생본 자동 생성기

### 1-2. 프로젝트 목적
사용자가 원본 PNG 로고 파일 1개를 선택하면, 버튼 한 번으로 다양한 파생 PNG 파일을 안정적으로 자동 생성하는 데스크톱 GUI 프로그램을 만든다.

### 1-3. 핵심 목표
- 초보자도 쉽게 사용할 수 있는 단순한 GUI 제공
- 원본 크기와 비율을 유지한 상태로 파생 이미지 생성
- 파일 선택, 저장 폴더 선택, 사용자 지정 색상 입력, 결과 저장까지 한 화면에서 처리
- 흰색 또는 단색에 가까운 배경을 최대한 자연스럽게 제거
- 실행 중 오류를 줄이고, 오류 발생 시 명확한 안내 메시지 제공

## 2. 범위 정의

### 2-1. 반드시 포함할 기능
- PNG 원본 파일 1개 선택
- 출력 폴더 선택
- 저장 파일명 기본 이름 입력
- 오늘 날짜를 포함한 저장 규칙 자동 적용
- 아래 PNG 결과물 자동 생성
  - 원본 복사본
  - 투명 배경 버전
  - 검정 버전
  - 흰색 버전
  - 회색조 버전
  - 반전 버전
  - 빨강 버전
  - 파랑 버전
  - 초록 버전
  - 노랑 버전
  - 사용자 지정 색상 버전
- 화면 내 간단 미리보기
- 예외 처리 및 상태 메시지 출력

### 2-2. 이번 단계에서 제외할 기능
- 일괄 다중 파일 처리
- 드래그 앤 드롭
- 실행 파일 배포 자동화
- 고급 배경 제거 AI 모델 적용
- 편집 이력 저장

## 3. 사용자 시나리오

### 3-1. 기본 사용 흐름
1. 사용자가 프로그램을 실행한다.
2. 원본 PNG 파일을 선택한다.
3. 결과 저장 폴더를 선택한다.
4. 저장 파일명 기본값을 입력한다.
5. 필요하면 사용자 지정 HEX 색상을 입력한다.
6. 미리보기 영역에서 주요 결과를 간단히 확인한다.
7. "자동 생성" 버튼을 누른다.
8. 지정 폴더에 날짜와 설명이 포함된 PNG 파일들이 저장된다.
9. 상태 메시지로 성공 여부와 저장 위치를 확인한다.

### 3-2. 파일명 저장 규칙
- 기본 규칙: `사용자입력파일명_YYYYMMDD_설명.png`
- 예시:
  - `brandlogo_20260310_original.png`
  - `brandlogo_20260310_transparent.png`
  - `brandlogo_20260310_black.png`
  - `brandlogo_20260310_color_custom.png`
- 사용자가 파일명 기본값을 입력하지 않으면 원본 파일명을 기본값으로 사용

## 4. 기능 계획

### 4-1. 파일 선택 기능
- PNG 파일만 선택 가능하도록 파일 선택 대화상자 제한
- PNG 이외의 파일이 들어오면 즉시 경고 메시지 표시
- 선택한 파일 경로를 화면에 표시

### 4-2. 출력 폴더 선택 기능
- 폴더 선택 대화상자로 저장 위치 지정
- 저장 위치 미선택 시 생성 버튼 실행 불가 또는 오류 메시지 표시
- 저장 폴더 경로를 화면에 표시

### 4-3. 자동 생성 기능
- 버튼 1회 클릭으로 모든 파생본 생성
- 생성 대상:
  - original
  - transparent
  - black
  - white
  - grayscale
  - invert
  - color_red
  - color_blue
  - color_green
  - color_yellow
  - color_custom
- 각 파일은 독립 저장하여 일부 실패 시 어떤 항목이 실패했는지 알려줌

### 4-4. 투명 배경 처리
- 기준 목표:
  - 흰색 또는 단색에 가까운 배경을 투명 처리
  - 로고 경계선 손상을 최소화
  - 가장자리의 하얀 테두리와 지저분한 노이즈를 줄임
- 구현 방향:
  - 이미지 가장자리 샘플링으로 배경색 후보 추정
  - 배경 유사도 기준으로 알파 마스크 생성
  - 임계값 보정으로 과도한 제거 방지
  - 가장자리 소프트닝으로 톱니 현상 완화

### 4-5. 색상 변형 기능
- 로고 형태를 유지한 채 전경색만 변경
- 투명 배경 결과를 기반으로 색상 치환
- 검정, 흰색, 사용자 지정 색상, 고정 색상 버전 생성
- 반전 버전은 RGB 색상을 반전하되 알파는 유지
- 회색조 버전은 명도 기반으로 변환

### 4-6. 미리보기 기능
- 최소 미리보기 대상:
  - 원본
  - 투명
  - 검정
  - 반전
  - 사용자 지정 색상 또는 대표 색상 1개
- 성능을 위해 미리보기는 축소본으로만 표시
- 원본 출력 파일은 절대 리사이즈하지 않음

### 4-7. 예외 처리 기능
- 파일 미선택
- 출력 폴더 미선택
- PNG가 아닌 파일 선택
- 잘못된 HEX 색상 입력
- 읽기 실패 또는 저장 실패
- 배경 제거가 불완전한 경우에도 중단하지 않고 가능한 결과물 저장

## 11. 용도별 폴더 구조와 자동 생성 PNG 정리 기준

이 프로그램은 단순히 PNG 파일만 여러 장 만드는 것이 아니라, 사용자가 결과물을 바로 찾고 바로 쓸 수 있도록 폴더 구조까지 함께 정리하는 방향으로 설계한다. 특히 초보자 입장에서는 파일명만 길게 나열되어 있으면 어떤 파일을 어디에 써야 하는지 헷갈리기 쉽기 때문에, 폴더 이름만 보아도 용도를 이해할 수 있는 구조가 필요하다.

### 11-1. 폴더 구조를 이렇게 나누는 이유

폴더 구조를 용도별로 나누는 이유는 단순 정리가 아니라 실제 사용 편의성을 높이기 위해서다.

- 파일을 잃어버리지 않기 위해 나눈다.
- 용도별로 바로 찾기 쉽게 하기 위해 나눈다.
- 같은 로고라도 플랫폼과 상황에 따라 써야 하는 버전이 달라지기 때문에 나눈다.
- 나중에 수정, 납품, 재사용이 쉬워지도록 나눈다.

예를 들어 명함에 넣는 로고와 인스타그램 스토리에 올리는 로고는 같은 브랜드 로고라도 쓰는 배경과 화면 조건이 다를 수 있다. 그래서 초보자도 "어디에 쓸 파일인지" 먼저 떠올리고 폴더를 열 수 있도록, 브랜드명 아래를 용도 중심으로 나누는 것이 가장 직관적이다.

또한 `00_원본` 폴더는 단순 보관 창고가 아니라, 모든 작업의 기준이 되는 기본 로고 파일을 모아두는 기준 폴더로 사용한다. 나중에 색상을 다시 바꾸거나, 새 플랫폼용 이미지를 추가로 만들거나, 외주 디자이너에게 기준 파일을 전달할 때 가장 먼저 확인해야 하는 폴더가 바로 이 위치다.

### 11-2. 상위 폴더별 설명

#### `00_원본`
- 폴더명: `00_원본`
- 언제 쓰는지: 가장 기준이 되는 로고 파일을 처음 정리할 때, 수정 작업을 다시 시작할 때, 새 파생본을 다시 만들 때 사용한다.
- 어떤 작업할 때 열어보면 되는지: 원본 파일 확인, 기준 색상 확인, 투명 기본형 확인, 흑백 기본형 확인, 반전 기본형 확인, 기본 색상 변형 확인이 필요할 때 열어본다.
- 초보자 설명: 이 폴더는 "가장 기본이 되는 로고 파일 보관함"이다. 실제 업로드용 파일을 찾는 곳이 아니라, 모든 작업의 출발점이 되는 기준 파일을 모아두는 곳이라고 이해하면 된다.

#### `01_인쇄`
- 폴더명: `01_인쇄`
- 언제 쓰는지: 명함, 메뉴판, 포스터, 스티커, 간판처럼 실제로 출력해서 쓰는 작업을 할 때 사용한다.
- 어떤 작업할 때 열어보면 되는지: 인쇄소 전달용 파일을 찾을 때, 종이 출력물이나 실물 제작물에 넣을 로고를 준비할 때 열어본다.
- 초보자 설명: "프린트해서 만드는 작업"에 들어가는 로고를 찾는 폴더라고 생각하면 된다.

#### `02_배달앱`
- 폴더명: `02_배달앱`
- 언제 쓰는지: 배민, 쿠팡이츠, 요기요 같은 배달 플랫폼에 브랜드 로고를 등록하거나 교체할 때 사용한다.
- 어떤 작업할 때 열어보면 되는지: 배달앱 관리자 페이지에 올릴 로고를 찾을 때, 앱별로 정리된 파일을 바로 확인할 때 열어본다.
- 초보자 설명: "배달앱에 올릴 로고는 여기 있다"라고 바로 이해할 수 있는 폴더다.

#### `03_네이버`
- 폴더명: `03_네이버`
- 언제 쓰는지: 네이버 플레이스, 블로그, 카페에 맞는 로고 파일이 필요할 때 사용한다.
- 어떤 작업할 때 열어보면 되는지: 네이버 서비스별 등록 이미지나 소개 이미지를 정리하거나 교체할 때 열어본다.
- 초보자 설명: 네이버 안에서 쓰는 로고 파일을 따로 모아둔 폴더라고 보면 된다.

#### `04_인스타그램`
- 폴더명: `04_인스타그램`
- 언제 쓰는지: 인스타그램 프로필, 피드, 스토리 등에 브랜드 로고를 적용할 때 사용한다.
- 어떤 작업할 때 열어보면 되는지: 계정 프로필 이미지 작업, 게시물 디자인, 스토리용 응용 이미지 작업을 할 때 열어본다.
- 초보자 설명: 인스타그램에서 실제로 보여줄 로고를 찾는 폴더다.

#### `05_웹사이트`
- 폴더명: `05_웹사이트`
- 언제 쓰는지: 홈페이지, 랜딩페이지, 쇼핑몰, 웹 공유 썸네일 등에 로고를 넣을 때 사용한다.
- 어떤 작업할 때 열어보면 되는지: 사이트 헤더에 넣을 로고, 브라우저 탭 아이콘용 이미지, 링크 공유용 썸네일을 찾을 때 열어본다.
- 초보자 설명: 웹사이트 안에서 쓰는 로고와 관련 이미지를 모아두는 폴더다.

#### `06_목업`
- 폴더명: `06_목업`
- 언제 쓰는지: 실제 업로드용 파일이 아니라, 시안 확인용 이미지나 적용 예시를 볼 때 사용한다.
- 어떤 작업할 때 열어보면 되는지: 간판에 얹어본 예시, 컵이나 포장지에 적용한 예시, SNS 썸네일 시안처럼 "미리 보기용" 결과물을 확인할 때 열어본다.
- 초보자 설명: 이 폴더는 실제 제출이나 업로드용보다는 "이렇게 보인다"를 확인하는 폴더라고 이해하면 된다.

### 11-3. 하위 폴더별 설명

#### `00_원본` 안의 보관용 하위 폴더

- `00_원본/원본그대로`
  - 수정 전 입력 PNG를 가장 기준이 되는 형태로 보관하는 폴더다.
  - "진짜 원본 파일이 어디 있지?"라고 찾게 될 때 가장 먼저 열어보는 폴더다.
- `00_원본/투명배경기본형`
  - 배경을 제거한 기본 로고를 보관하는 폴더다.
  - 다른 디자인 작업에서 배경 없는 로고가 필요할 때 기준 파일로 꺼내 쓴다.
- `00_원본/흑백기본형`
  - 검정, 흰색, 회색조처럼 기본 흑백 계열 로고를 모아두는 폴더다.
  - 단색 인쇄나 간단한 시안 작업을 시작할 때 먼저 확인하기 좋다.
- `00_원본/반전기본형`
  - 어두운 배경 위에 올릴 때 쓰는 반전 기본형을 보관하는 폴더다.
  - 검은 배경, 진한 색 배경, 야간 느낌의 디자인 시안에 넣을 때 기준 파일로 사용한다.
- `00_원본/색상변형기본형`
  - 빨강, 파랑, 초록, 노랑, 사용자 지정 색상처럼 응용 색상 버전을 모아두는 폴더다.
  - 이벤트 디자인이나 확장 브랜드 컬러 테스트를 할 때 먼저 확인하면 된다.

  - 음식점, 카페, 매장 메뉴판 디자인 작업을 할 때 열어본다.
- `01_인쇄/포스터`
  - 행사 포스터, 매장 안내 포스터, 홍보 포스터에 넣을 로고를 찾는 폴더다.
  - 배경색이 큰 포스터 작업에서 어떤 버전이 잘 보이는지 고를 때 열어본다.
- `01_인쇄/스티커`

### 11-4. 각 실사용 폴더 내부 공통 구조

여기서 말하는 "실사용 폴더"는 실제로 사용처가 정해져 있는 폴더를 뜻한다. 예를 들어 `01_인쇄/명함`, `02_배달앱/배민`, `03_네이버/플레이스`, `04_인스타그램/스토리`, `05_웹사이트/헤더로고` 같은 폴더가 여기에 해당한다.

이 실사용 폴더들 안에는 아래 4개의 하위 폴더가 공통으로 자동 생성된다.

- `투명배경`
  - 배경 없이 로고만 필요한 경우에 쓰는 PNG를 모아두는 폴더다.
  - 예: `logo_20260310_transparent.png`
- `흑백`
  - 단색 인쇄나 단순한 적용이 필요한 경우에 쓰는 PNG를 모아두는 폴더다.
  - 예: `logo_20260310_black.png`, `logo_20260310_white.png`, `logo_20260310_grayscale.png`
- `반전`
  - 어두운 배경 위에 올릴 때 쓰는 밝은 계열의 반전 PNG를 모아두는 폴더다.
  - 예: `logo_20260310_invert.png`
- `색상변형`
  - 브랜드 확장, 이벤트용, 응용 버전이 필요할 때 쓰는 PNG를 모아두는 폴더다.
  - 예: `logo_20260310_color_red.png`, `logo_20260310_color_blue.png`, `logo_20260310_color_green.png`, `logo_20260310_color_yellow.png`, `logo_20260310_color_custom.png`

이 구조는 모든 실사용 폴더에 동일하게 들어간다. 즉 `01_인쇄/명함` 안에도, `02_배달앱/배민` 안에도, `03_네이버/플레이스` 안에도, `04_인스타그램/프로필` 안에도, `05_웹사이트/공유썸네일` 안에도 같은 방식으로 `투명배경`, `흑백`, `반전`, `색상변형` 폴더가 자동으로 생긴다.

예를 들면 아래와 같은 방식으로 이해하면 된다.

- `01_인쇄/명함/투명배경/`
- `01_인쇄/명함/흑백/`
- `01_인쇄/명함/반전/`
- `01_인쇄/명함/색상변형/`
- `02_배달앱/배민/투명배경/`
- `02_배달앱/배민/흑백/`
- `02_배달앱/배민/반전/`
- `02_배달앱/배민/색상변형/`

반대로 원본 그대로 복사한 기준 파일은 실사용 폴더마다 중복 보관하지 않고, `00_원본/원본그대로` 아래에 모아두는 방식을 기본으로 한다. 이렇게 해야 기준 파일이 여기저기 흩어지지 않고, 나중에 수정할 때도 가장 먼저 확인해야 할 위치가 분명해진다.

이 섹션은 앞서 정리한 폴더 구조 안에서, 각 폴더에 어떤 PNG 사이즈를 자동 생성해야 하는지 초보자도 바로 이해할 수 있도록 정리한 기준이다. 같은 로고라도 올리는 위치에 따라 잘리는 방식과 권장 비율이 다르고, 인쇄물과 플랫폼 이미지의 쓰임새도 다르기 때문에, 폴더 구조와 함께 사이즈 기준을 따로 정리해야 실제 사용이 쉬워진다.

### 사이즈 기준을 따로 두는 이유

- 같은 로고라도 업로드 위치마다 잘리는 방식과 권장 비율이 다르기 때문이다.
- 인쇄물과 플랫폼 이미지는 쓰임새와 해상도 기준이 다르기 때문이다.
- 초보자도 폴더만 보고 바로 꺼내 쓸 수 있게 하려면, 폴더 이름뿐 아니라 생성 크기까지 함께 정리되어 있어야 하기 때문이다.

예를 들어 `02_배달앱/배민`에 들어가는 로고와 `04_인스타그램/스토리`에 들어가는 로고는 같은 PNG라도 필요한 비율과 노출 방식이 다르다. 따라서 자동 생성 프로그램은 단순히 색상 버전만 만드는 것이 아니라, 용도에 맞는 크기 프리셋까지 같이 만들어주는 방식으로 설계하는 것이 실사용에 더 적합하다.

### 공식 기준과 실무 권장 프리셋을 구분하는 방식

이 섹션에서는 사이즈 기준을 두 가지로 나누어 표기한다.

- `공식 기준`
  - 공개된 도움말, 고객센터, 개발자 가이드, 운영 가이드에서 확인된 값 또는 공개적으로 일관되게 안내되는 규격을 뜻한다.
- `실무 권장 프리셋`
  - 공개 공식값이 명확하지 않거나, 자동 생성 프로그램에서 바로 쓰기 좋게 정리한 내부 제작용 기준을 뜻한다.

따라서 문서 안에서 `공식 기준`이라고 적힌 값은 공개 가이드 기준으로 이해하면 되고, `실무 권장 프리셋`이라고 적힌 값은 자동 생성 프로그램이 미리 만들어두면 실제 작업이 편해지는 제작 기준으로 이해하면 된다. 두 기준은 섞어서 쓰지 않고, 각 항목에서 명확히 분리해 적는다.

| 정사각 보조형 | 1200x1200 | 실무 권장 프리셋 | SNS/메신저 보조 공유 이미지 |

### `06_목업` 사이즈 기준

`06_목업`은 업로드용이 아니라 적용 예시 확인용 폴더다. 즉 실제 등록용 PNG를 넣는 폴더가 아니라, 로고를 명함, 메뉴판, 포스터, 스티커, 간판 등에 올려보았을 때 어떻게 보이는지 확인하는 시안용 폴더다.

- 공식 기준
  - 해당 없음
- 실무 권장 프리셋

| 목업 항목 | 자동 생성 사이즈 | 구분 | 용도 |
| --- | --- | --- | --- |
| 명함 목업 | 1600x1000 | 실무 권장 프리셋 | 명함 시안 확인 |
| 메뉴판 목업 | 1600x2200 | 실무 권장 프리셋 | 메뉴판 시안 확인 |
| 포스터 목업 | 2000x2800 | 실무 권장 프리셋 | 포스터 시안 확인 |
| 스티커 목업 | 1600x1600 | 실무 권장 프리셋 | 스티커 시안 확인 |
| 간판 목업 | 2400x1600 | 실무 권장 프리셋 | 간판 적용 예시 확인 |

### 각 실사용 폴더 안의 파생 구조와 사이즈 적용 방법

각 실사용 폴더 안에는 아래 하위 폴더가 공통으로 존재하고, 각 하위 폴더 안에는 해당 용도의 사이즈로 PNG가 자동 생성된다.

- `투명배경`
  - 배경 없는 PNG를 해당 용도 사이즈에 맞춰 저장한다.
- `흑백`
  - 검정, 흰색, 회색조 PNG를 해당 용도 사이즈에 맞춰 저장한다.
- `반전`
  - 어두운 배경 대응용 반전 PNG를 해당 용도 사이즈에 맞춰 저장한다.
- `색상변형`
  - 빨강, 파랑, 초록, 노랑, 사용자 지정 색상 PNG를 해당 용도 사이즈에 맞춰 저장한다.

예를 들어 `02_배달앱/배민/투명배경/` 안에는 아래와 같은 파일이 들어간다.

- `baemin_logo_170x170.png`
- `baemin_logo_hd_1020x1020.png`
- `baemin_store_1200x1200.png`
- `baemin_menu_1280x960.png`

또한 `04_인스타그램/피드/색상변형/` 안에는 아래와 같은 파일이 들어간다.

- `instagram_feed_square_1080x1080.png`
- `instagram_feed_vertical_1080x1350.png`

같은 방식으로 `03_네이버/플레이스/흑백/` 안에는 `naver_place_main_1200x750.png`, `naver_place_thumb_720x720.png` 같은 파일이 들어가고, `05_웹사이트/파비콘/반전/` 안에는 `favicon_48x48.png`, `favicon_96x96.png`, `favicon_192x192.png`, `favicon_512x512.png` 같은 파일이 들어가도록 정리한다.

즉 프로그램은 먼저 "어디에 쓸 폴더인지"를 기준으로 위치를 나누고, 그다음 "어떤 색상 버전인지"에 따라 `투명배경`, `흑백`, `반전`, `색상변형`으로 나눈 뒤, 마지막으로 그 안에 실제 용도별 해상도 파일을 생성하는 흐름으로 동작해야 한다.
