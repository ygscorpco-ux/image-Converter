# PNG 로고 파생본 자동 생성기 기획서

## 1. 문서 목적
본 문서는 실제 구현을 위한 기술 방향, 코드 구조, 처리 흐름, 저장 규칙을 정의한다. 목표는 복잡한 구조보다 안정성과 유지보수성을 우선한 단일 실행 파일 형태의 Python GUI 프로그램을 설계하는 것이다.

## 2. 개발 목표 요약
- Python 기반의 데스크톱 GUI 프로그램 제작
- Tkinter를 우선 사용하여 초보자 실행 편의성 확보
- Pillow 기반 이미지 처리
- 한 파일 실행형 구조를 우선 적용
- 추후 확장 가능한 함수 중심 구조 설계

## 3. 기술 스택

### 3-1. 언어 및 런타임
- Python 3.11 이상 권장

### 3-2. GUI
- Tkinter
- 선택 이유:
  - Python 기본 포함으로 설치 부담이 적음
  - 단순한 툴성 프로그램 제작에 적합
  - 초보자 환경에서 실행 안정성이 높음

### 3-3. 이미지 처리
- Pillow
- 사용 목적:
  - PNG 로드 및 저장
  - RGBA 변환
  - 회색조 처리
  - 색상 치환
  - 반전 처리
  - 알파 마스크 생성
  - 미리보기 축소본 생성

### 3-4. 선택적 보조 라이브러리
- numpy
- 사용 조건:
  - 배경 제거 마스크 계산을 조금 더 안정적으로 처리하고 싶을 때만 사용
  - 필수 의존성은 아님

## 4. 실행 환경

### 4-1. 대상 운영체제
- Windows 우선

### 4-2. 배포 형태
- 1차 목표:
  - `python main.py` 로 실행 가능한 형태

## 5. 파일 구조 기획

이번 버전은 한 파일 실행형을 우선하므로 아래 구조를 기준으로 한다.

```text
image Converter/
├─ main.py
├─ 개발계획서.md
└─ 기획서.md
```

### 5-1. 한 파일 내부 논리 구조
`main.py` 안에서 아래 순서로 구성한다.

1. 상수 정의
2. 유틸리티 함수
3. 이미지 처리 함수
4. 파일명 생성 함수
5. 미리보기 생성 함수
6. GUI 이벤트 처리 함수
7. Tkinter 앱 초기화 및 실행

## 6. 주요 기능 설계

### 6-1. 입력값
- 원본 PNG 파일 경로
- 출력 폴더 경로
- 저장 파일명 기본 문자열
- 사용자 지정 HEX 색상값

### 6-2. 출력값
- 아래 파일들을 지정 폴더에 저장
  - `{base}_{date}_original.png`
  - `{base}_{date}_transparent.png`
  - `{base}_{date}_black.png`
  - `{base}_{date}_white.png`
  - `{base}_{date}_grayscale.png`
  - `{base}_{date}_invert.png`
  - `{base}_{date}_color_red.png`
  - `{base}_{date}_color_blue.png`
  - `{base}_{date}_color_green.png`
  - `{base}_{date}_color_yellow.png`
  - `{base}_{date}_color_custom.png`

참고:
- 요구사항 목록의 설명에는 반전 버전이 포함되어 있으므로 실제 구현에서는 `invert` 파일도 생성 대상으로 포함하는 것이 안전하다.
- 사용자가 검정/흰색 버전을 별도 요구했으므로 `black`, `white`도 명확히 분리 저장한다.

## 7. 저장 파일명 규칙

### 7-1. 규칙
- 패턴: `기본파일명_YYYYMMDD_설명.png`
- 예시:
  - `mylogo_20260310_original.png`
  - `mylogo_20260310_transparent.png`
  - `mylogo_20260310_color_blue.png`

### 7-2. 기본파일명 결정 방식
- 사용자가 입력한 값이 있으면 우선 사용
- 없으면 원본 파일의 확장자 제거한 이름 사용
- 파일명에 사용할 수 없는 문자는 `_`로 치환

## 8. 이미지 처리 설계

### 8-1. 공통 원칙
- 원본 가로/세로 크기 유지
- 비율 유지
- 저장 형식은 모두 PNG
- 처리 전 내부적으로 `RGBA` 로 통일

### 8-2. 원본 복사본
- 입력 이미지를 그대로 다시 저장
- 메타 수준의 단순 재저장보다 픽셀 유지 중심으로 처리

### 8-3. 투명 배경 생성
- 예상 입력:
  - 흰 배경
  - 밝은 단색 배경
  - 이미 투명 배경인 PNG
- 처리 순서:
  1. 이미지 가장자리 픽셀을 샘플링하여 배경 대표색 추정
  2. 각 픽셀과 배경색의 거리 계산
  3. 유사도가 높은 영역을 투명화
  4. 경계값 주변은 반투명 처리로 자연스러운 가장자리 유지
  5. 이미 알파 채널이 있으면 기존 알파를 존중
- 기대 효과:
  - 단색 배경 제거
  - 하얀 외곽선 감소

### 8-4. 검정/흰색/고정색/사용자색 생성
- 투명 버전을 기준 이미지로 사용
- 알파는 그대로 유지
- RGB만 지정 색상으로 치환
- 전경 부분은 로고 형상을 유지한 채 색상만 바뀌도록 설계

### 8-5. 회색조 생성
- Pillow의 grayscale 변환 후 RGBA 복원
- 기존 알파 채널 유지

### 8-6. 반전 생성
- 알파 채널을 제외한 RGB만 반전
- 완전 투명 영역은 그대로 유지

### 8-7. 미리보기 생성
- 화면 표시용 축소본만 생성
- 썸네일 크기는 일정하게 제한
- 실제 저장본은 절대 축소하지 않음

## 9. GUI 설계

### 9-1. 화면 요소
- 제목 라벨
- 원본 파일 선택 버튼 + 경로 표시 라벨
- 출력 폴더 선택 버튼 + 경로 표시 라벨
- 저장 파일명 입력칸
- 사용자 지정 HEX 입력칸
- 미리보기 버튼
- 자동 생성 버튼
- 상태 메시지 라벨
- 미리보기 썸네일 프레임

### 9-2. 버튼 정책
- 파일 선택: PNG만 허용
- 폴더 선택: 존재하는 디렉터리만 허용
- 미리보기: 입력 검증 후 수행
- 자동 생성: 필수 입력 미완성 시 진행 금지

### 9-3. 상태 메시지 예시
- `PNG 파일을 선택해주세요.`
- `출력 폴더를 선택해주세요.`
- `HEX 색상 형식이 올바르지 않습니다. 예: #1B4797`
- `11개 파일 생성 완료`
- `일부 파일 저장에 실패했습니다. 상태를 확인해주세요.`

- 출력 폴더 미선택
- PNG 외 확장자
- 잘못된 HEX 형식

## 11. 코드 구조 기획

`main.py` 내부에서 다음과 같은 함수 구성을 권장한다.

### 11-1. 유틸리티 함수
- `is_valid_hex_color(value)`
- `sanitize_filename(value)`
- `build_output_name(base_name, suffix, date_text)`

### 11-2. 이미지 처리 함수
- `load_image(path)`
- `detect_background_color(image)`
- `remove_background(image)`
- `recolor_with_alpha(image, rgb_color)`
- `create_grayscale(image)`
- `create_inverted(image)`
- `generate_all_variants(image, custom_color)`

### 11-3. GUI 처리 함수
- `choose_input_file()`
- `choose_output_folder()`
- `refresh_preview()`
- `run_generation()`
- `update_status(message, is_error=False)`

### 11-4. 앱 구성 방식
- 클래스 1개로 GUI 상태를 묶는 방식이 적합
- 예시:
  - `class LogoConverterApp:`
- 이유:
  - Tkinter 위젯 상태 관리가 쉬움
  - 경로, 입력값, 썸네일 참조를 한 곳에서 관리 가능

## 12. 처리 흐름 설계

```text
프로그램 시작
  -> 파일 선택
  -> 출력 폴더 선택
  -> 파일명/HEX 입력
  -> 입력 검증
  -> 원본 이미지 로드
  -> 투명 배경 버전 생성
  -> 파생 이미지 일괄 생성
  -> 파일명 규칙 적용 후 저장

```

```text

├─ main.py
├─ image_ops.py
├─ ui.py
├─ utils.py
```

## 16. 결론

이 기획서는 Tkinter + Pillow 기반의 안정적인 단일 파일 프로그램을 만드는 데 필요한 기술적 기준을 정의한다. 이 구조를 따르면, 사용자는 로컬 환경에서 PNG 로고를 선택하고 버튼 한 번으로 다양한 파생 PNG 파일을 생성할 수 있다.
